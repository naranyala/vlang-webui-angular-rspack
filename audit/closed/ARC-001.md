# Architecture Finding: ARC-001

## Tight Coupling to WebUI Library

**Severity:** HIGH  
**Impact:** Difficult to test, replace, or upgrade window management  
**Status:** [DONE] **RESOLVED**

---

## Resolution

### Fixed In
- `src/window_manager.v` - Window management abstraction

### Changes Made

1. **Created IWindowManager interface:**
```v
pub interface IWindowManager {
    mut:
        init() !
        bind(event string, handler fn (&ui.Event) string)
        show(page string) !
        wait()
        close()
        set_title(title string)
        get_size() (int, int)
        resize(width int, height int)
}
```

2. **Implemented WebUIWindowManager:**
```v
pub struct WebUIWindowManager {
pub mut:
    window          ui.Window
    initialized     bool
    title           string
    width           int
    height           int
    handlers        map[string]string
}

pub fn (mut m WebUIWindowManager) bind(event string, handler fn (&ui.Event) string) {
    m.window.bind(event, handler)
    m.handlers[event] = 'registered'
}
```

3. **Created MockWindowManager for testing:**
```v
pub struct MockWindowManager {
pub mut:
    initialized     bool
    bound_events    map[string]bool
    shown_pages     []string
    wait_called     bool
    close_called    bool
}

pub fn (mut m MockWindowManager) bind(event string, handler fn (&ui.Event) string) {
    m.bound_events[event] = true
}
```

4. **Updated main.v to use abstraction:**
```v
// ARC-001: Use window manager abstraction
mut window_mgr := window_manager.new_webui_window_manager()
window_mgr.init() or {
    vlog('ERROR: Failed to initialize window manager')
    return
}
```

### Benefits

- **Testability:** Can use MockWindowManager for unit tests
- **Swappability:** Easy to replace WebUI with alternative library
- **Separation of Concerns:** Window management isolated to one module
